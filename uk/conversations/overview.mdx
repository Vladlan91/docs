## Огляд

Сторінка Розмов — це єдина панель для всіх взаємодій між вашими AI агентами та клієнтами, незалежно від того, який агент обробляв розмову чи який канал використовував відвідувач (віджет, телефон, Telegram, WhatsApp тощо).

Ця сторінка працює однаково для всіх режимів роботи агента. В **активному** режимі ви бачите розмови, які вів AI. В **пасивному** режимі — розмови, які вела команда, з доданими даними аудиту та аналізу від агента. В **гібридному** режимі обидва типи відображаються в одному єдиному вигляді.

Перейдіть до **Розмови** у бічному меню, щоб відкрити цю сторінку.

## Таблиця розмов

Таблиця відображає всі розмови з усіх ваших агентів, відсортовані за датою (найновіші першими). Кожен рядок показує:

| Колонка | Опис |
|---------|------|
| **Agent** | Ім'я агента та компанії, з іконкою каналу |
| **Session** | Ідентифікатор сесії відвідувача |
| **Channel** | Бейдж каналу зв'язку — Widget, Web, Telegram, WhatsApp, VoIP, Messenger, CRM |
| **Status** | Поточний стан — Active (активна), Closed (завершена) або Archived (архівована) |
| **Messages** | Загальна кількість повідомлень у розмові |
| **Rating** | Оцінка клієнта від 1 до 5, або `—` якщо не оцінено |
| **Date** | Коли розпочалась розмова |
| **Actions** | Кнопка видалення |

Натисніть на заголовок будь-якої колонки (Agent, Status, Messages, Rating, Date), щоб змінити порядок сортування.

### Фільтрація

Натисніть кнопку **Filter**, щоб додати фільтри-чіпи. Кожен фільтр звужує результати:

| Фільтр | Варіанти |
|--------|----------|
| **Status** | Active, Closed, Archived |
| **Channel** | Web, Widget, Telegram, WhatsApp, VoIP, Messenger, CRM |
| **Agent** | Вибір з усіх ваших агентів |
| **Min Rating** | 1+, 2+, 3+, 4+, 5 |
| **Date From** | Календарний вибір дати |
| **Date To** | Календарний вибір дати |

Декілька фільтрів працюють разом (логіка AND). Видаліть фільтр, натиснувши `×` на його чіпі.

### Пошук

Поле пошуку виконує пошук без урахування регістру одночасно за трьома вимірами:
- **Session ID** — часткове співпадіння
- **Ім'я агента** — часткове співпадіння
- **Вміст повідомлень** — пошук всередині всіх повідомлень розмови

## Деталі розмови

Натисніть на будь-який рядок, щоб відкрити панель деталей — бічний offcanvas, що висувається справа. Панель має дві колонки: основна область контенту (ліва) та бічна панель метаданих (права).

### Аудіо плеєр

Для голосових розмов (VoIP, голосовий віджет) у верхній частині панелі з'являється вбудований аудіо плеєр. Можливості:

- **Play/Pause** з візуальною хвильовою формою
- **Швидкість відтворення** — перемикання між 0.5×, 1.0×, 1.5×, 2.0×
- **Перемотка** вперед і назад на 5 секунд
- **Перехід** натисканням на будь-яке місце хвильової форми
- **Автоматичне продовження** — коли одне повідомлення завершується, наступне аудіо повідомлення відтворюється автоматично

Аудіо файли зберігаються на сервері для кожної розмови та завантажуються на вимогу при відкритті панелі деталей.

Для текстових розмов плеєр замінюється індикатором текстового режиму.

### Бічна панель метаданих

Завжди видима у правій частині панелі:

| Поле | Опис |
|------|------|
| **Date** | Коли розпочалась розмова |
| **Text-only** | Чи є у розмові голосові повідомлення |
| **Channel** | Канал зв'язку |
| **Duration** | Загальна тривалість розмови, або "Ongoing" якщо ще активна |
| **Messages** | Загальна кількість повідомлень |
| **Tokens** | Загальна кількість LLM токенів (prompt + completion) |
| **Rating** | Оцінка клієнта зірками (1–5) або "No rating" |
| **Agent** | Який агент обробляв розмову |

## Три вкладки

Основна область контенту має три вкладки:

### Overview

Швидке зведення розмови:

- **Call status** — Active, Closed або Archived
- **How the call ended** — причина завершення або "Completed"
- **Sentiment** — AI-аналіз тону розмови (positive, negative, neutral) або "Not analyzed"
- **Session ID** — унікальний ідентифікатор сесії відвідувача

### Transcription

Повна історія повідомлень, відображена як чат-бульбашки. Показуються лише повідомлення користувача та асистента — системні та інструментальні повідомлення приховані. Повідомлення користувача відображаються як звичайний текст; повідомлення асистента підтримують форматування (жирний, посилання, списки, таблиці, блоки коду, зображення).

<Note>
  Повідомлення пагінуються по 50 на сторінку. Для довгих розмов спочатку відображаються найновіші повідомлення.
</Note>

### Client Data

Найдетальніша вкладка — поєднує метадані розмови, витягнуту AI пам'ять та пов'язану аналітику трекінгу.

#### Conversation

Інформаційна сітка з 8 комірок: Channel, Status, Duration, Messages, Sentiment, Converted (Yes/No), Session ID та Start date.

#### Memory

<Tip>
  Поля пам'яті витягуються AI агентом під час розмови на основі інструкцій у системному промпті. Вони можуть включати будь-які дані, які агент налаштований збирати — ім'я клієнта, телефон, email, номер замовлення, інтерес до продукту, статус кваліфікації тощо.
</Tip>

Якщо розмова містить витягнуті дані пам'яті, вони відображаються у вигляді сітки карток ключ-значення. Ключі та значення відображаються точно так, як їх записав AI. Дані пам'яті зберігаються як JSON у записі розмови та оновлюються в реальному часі, коли агент витягує нову інформацію.

#### Session (з Tracker)

Якщо розмова відвідувача пов'язана з трекінговою сесією (через віджет на сайті), панель показує аналітику сесії:

- **Device** — десктоп, мобільний або планшет
- **Browser** та **OS**
- **Country** та **City** — геолокація відвідувача
- **Landing page** — перший URL, який відкрив відвідувач
- **Traffic source** — звідки прийшов відвідувач
- **UTM source** та **UTM medium** — параметри атрибуції кампанії
- **Session duration** — загальний час на сайті
- **Pages visited** — кількість переглянутих сторінок
- **Bounce** — чи пішов відвідувач після однієї сторінки

#### Pageviews

Якщо пов'язана сесія має перегляди сторінок, кожна сторінка показується з її назвою, проведеним часом та відсотком прокрутки.

#### Events

Не-тригерні події, зафіксовані під час сесії — кліки, відправки форм, користувацькі події — з їх назвами, типами та часовими мітками.

#### Triggers

Тригерні події, що зібрали дані форм або іншу структуровану інформацію. Кожен тригер показує назву, тип дії, часову мітку та розгортний JSON-блок із зафіксованими полями даних. Кнопка копіювання дозволяє скопіювати сирий JSON.

## Оцінка

### Оцінка клієнта

Відвідувачі можуть оцінити свою розмову від 1 до 5 зірок через віджет. Оцінка та необов'язковий коментар відображаються як у таблиці, так і в панелі деталей. За потреби ви можете видалити оцінку клієнта з панелі деталей.

### Використання токенів

Кожна розмова відстежує використання LLM токенів — prompt tokens (вхідні) та completion tokens (вихідні). Загальна сума показується у бічній панелі метаданих. Це допомагає моніторити витрати на AI для кожної розмови.

## Видалення розмов

Натисніть іконку кошика у рядку таблиці, щоб видалити розмову. Перед видаленням з'являється діалог підтвердження. Видалення розмови остаточно видаляє:

- Всі повідомлення та їх вміст
- Всі аудіо файли, збережені на сервері
- Будь-які пов'язані оцінки

<Warning>
  Видалення незворотне. Аудіозаписи та історію повідомлень неможливо відновити.
</Warning>
