Дашборд має чотири секції:

### Зведення плану

Показує назву поточного плану, ціну та дату наступного платежу. Звідси можна підвищити план, керувати підпискою або відкрити Stripe Customer Portal.

### Використання ресурсів

Шість прогрес-барів, що показують споживання в реальному часі:

| Ресурс | Що вимірює |
|--------|-----------|
| **Standard Tokens** | Токени, спожиті стандартними моделями (GPT-4o Mini, Haiku, Flash, Llama 8B) |
| **Premium Tokens** | Токени, спожиті преміум моделями (GPT-4o, Sonnet, Gemini Pro, Llama 70B) |
| **STT Minutes** | Час транскрипції мовлення в текст |
| **TTS Characters** | Символи синтезу тексту в мовлення |
| **Embedding Tokens** | Токени для тренування бази знань (розбивка на чанки + генерація векторів) |
| **Storage** | Загальне файлове сховище по всіх компаніях |

Кожен бар кольорово кодований:
- **Зелений** — нижче 80% використання
- **Помаранчевий** — 80–89% (зона попередження)
- **Червоний** — 90%+ (критична зона)

### Ліміти платформи

Чотири лічильники, що показують скільки ресурсів створено vs. максимум плану:

- **Компанії** — напр., 2 / 3
- **AI-агенти** — напр., 4 / 5
- **Документи бази знань** — напр., 45 / 100
- **Розмови** — напр., 1 200 / 5 000

### Події використання

Детальна фільтрована таблиця кожної платної дії. Кожна подія записує:

| Поле | Опис |
|------|------|
| **Date** | Коли подія відбулась |
| **Company** | Агент якої компанії ініціював |
| **Event** | Тип дії — чат, голос, embedding |
| **Resource** | Тип спожитих токенів |
| **Channel** | Widget, VoIP, Telegram тощо |
| **Model** | Яка LLM-модель використовувалась |
| **Amount** | Кількість спожитих токенів, хвилин або символів |
| **Cost** | Орієнтовна вартість у USD |

Фільтруйте за діапазоном дат, компанією, типом ресурсу або каналом. Таблиця пагінована та завантажується асинхронно.

## Як працюють токени

Кожна AI-взаємодія має дві частини:

- **Вхідні токени** — ваше повідомлення + системний промпт + контекст бази знань (RAG-чанки) + історія розмови + описи інструментів
- **Вихідні токени** — відповідь агента

Обидва типи зараховуються до вашої квоти. Загальна кількість за взаємодію залежить від:

1. **Довжина системного промпту** — довші промпти споживають більше вхідних токенів на повідомлення
2. **Контекст бази знань** — більше RAG-чанків = більше вхідних токенів
3. **Історія розмови** — довші розмови накопичують контекст
4. **Довжина відповіді** — контролюється повзунком Response Length
5. **Виклики інструментів** — кожен раунд виклику споживає додаткові токени

### Типи токенів

| Тип токенів | Моделі | Профіль вартості |
|-------------|--------|-----------------|
| **Standard** | GPT-4o Mini, Claude 3.5 Haiku, Gemini 2.0 Flash, Llama 3.1 8B | Нижча вартість, швидші відповіді |
| **Premium** | GPT-4o, Claude 3.5 Sonnet, Gemini 2.5 Pro, Llama 3.3 70B | Вища вартість, краща якість |
| **Embedding** | text-embedding-ada-002 | Використовується лише під час тренування бази знань |

<Tip>
  Для оптимізації використання токенів: тримайте системні промпти стислими, обмежте кількість RAG-чанків на запит (RAG Settings → Chunk Limit), використовуйте стандартні моделі для простих задач, а преміум моделі резервуйте для складних відповідей.
</Tip>

## Використання голосу

Голосові взаємодії споживають два окремих ресурси:

- **STT (Speech-to-Text)** — вимірюється в хвилинах. Кожне голосове повідомлення транскрибується, і тривалість (округлена вгору до найближчої хвилини) списується з квоти.
- **TTS (Text-to-Speech)** — вимірюється в символах. Текстова відповідь агента синтезується в аудіо, і кількість символів списується.

Один голосовий поворот розмови зазвичай споживає і STT хвилини (аудіо відвідувача), і TTS символи (голосова відповідь агента).

## Використання сховища

Сховище споживається при завантаженні файлів у Файловий менеджер:

- **Документи** — PDF, Word, Excel, TXT
- **Фото** — JPG, PNG
- **Відео** — MP4, WebM, MOV
- **Аудіозаписи** — файли аудіо розмов

Використання сховища відстежується двонаправлено — завантаження збільшує використання, видалення файлів зменшує. Дашборд показує поточний загальний обсяг.

## Сповіщення

Revol моніторить ваше використання та надсилає попередження перед досягненням лімітів:

| Поріг | Рівень | Сповіщення |
|-------|--------|-----------|
| **80%** | Попередження | Email + in-app: "Ви використали 80% вашого [ресурсу]. Розгляньте оновлення або придбання пакету." |
| **90%** | Критичне | Email + in-app: "Ви використали 90% вашого [ресурсу]. Запити будуть заблоковані при 100%." |

Кожне попередження спрацьовує **один раз за білінговий період** на тип ресурсу. Критичні попередження мають пріоритет — якщо використання стрибнуло з 75% до 92%, надсилається лише критичне сповіщення.

## Білінговий період

- **Безкоштовний план** — щоденний період. Використання токенів скидається щодня опівночі UTC.
- **Платні плани** — щомісячний період. Використання скидається у річницю дати підписки (напр., якщо ви підписались 15 березня, наступне скидання — 15 квітня).

Додаткові пакети незалежні від білінгового періоду — вони мають власний 3-місячний термін дії від дати придбання.
