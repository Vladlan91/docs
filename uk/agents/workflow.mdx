## Огляд

Редактор Workflow — це візуальний інструмент на основі вузлів для проєктування логіки розмов. Перетягуйте та з'єднуйте вузли для створення складних потоків розмов.

## Типи вузлів

### Вузол Start
Точка входу для кожного workflow. Кожен workflow має рівно один вузол Start.

### Вузол LLM
Надсилає повідомлення до налаштованої AI моделі та повертає відповідь.

**Налаштування:**
- Temperature (0.0 - 1.0)
- Max tokens
- Користувацьке перевизначення system prompt

### Вузол Condition
Розгалужує workflow на основі умов. Підтримує:
- Збіг за ключовими словами
- Аналіз тональності
- Користувацькі вирази

### Вузол Tool Call
Виконує зовнішню функцію або API виклик. Результати передаються назад у контекст розмови.

### Вузол Voice Call
Обробляє голосову взаємодію — розпізнавання та синтез мовлення.

### Вузол End
Завершує розмову з необов'язковим заключним повідомленням.

## З'єднання вузлів

Натисніть на вихідний порт вузла та перетягніть до вхідного порту іншого вузла, щоб створити з'єднання. Кожне з'єднання представляє можливий шлях розмови.

## Найкращі практики

<Warning>
  Завжди переконуйтесь, що існує шлях від Start до End. Недосяжні вузли ніколи не виконуватимуться.
</Warning>

- Зберігайте workflow простими — складні потоки важче відлагоджувати
- Використовуйте вузли Condition для обробки граничних випадків
- Тестуйте кожен шлях окремо перед публікацією
