## Огляд

Шість сервісів Google доступні через єдиний SDK (`google/apiclient`) та спільний потік OAuth. Підключіться один раз зі своїм акаунтом Google — і ваші AI-агенти отримають доступ до електронної пошти, документів, таблиць, файлів, календаря та відеозустрічей.

## Підтримувані сервіси

| Сервіс | Інструменти | Опис |
|---|---|---|
| **Gmail** | 6 | Надсилання листів, читання вхідних, пошук, відповідь, створення чернеток |
| **Google Docs** | 5 | Створення, читання, додавання, пошук та експорт документів |
| **Google Sheets** | 7 | Читання/запис діапазонів, додавання рядків, пошук, створення таблиць |
| **Google Drive** | 8 | Перегляд, пошук, завантаження, вивантаження, надання доступу до файлів і папок |
| **Google Calendar** | 6 | Створення подій, перевірка доступності, пошук вільних слотів |
| **Google Meet** | 1 | Створення посилань на відеозустрічі |

## Автентифікація

Усі інтеграції Google Workspace використовують **OAuth 2.0** з офлайн-доступом:

<Steps>
  <Step title="Додайте інтеграцію">
    Перейдіть до **Інтеграції** → **Додати інтеграцію** → оберіть будь-який сервіс Google (наприклад, Gmail).
  </Step>
  <Step title="Екран згоди OAuth">
    Натисніть "Підключити Google". Вас буде перенаправлено на екран згоди Google, де ви авторизуєте доступ до конкретного сервісу.
  </Step>
  <Step title="Збереження токенів">
    Revol отримує access token та refresh token. Обидва зашифровані та зберігаються в базі даних. Refresh token забезпечує довготривалий доступ — Revol автоматично оновлює прострочені access token.
  </Step>
  <Step title="Створення інструментів">
    Після успішного OAuth інструменти агента створюються автоматично відповідно до типу інтеграції.
  </Step>
</Steps>

### OAuth Scopes

Кожен сервіс запитує лише необхідні scopes:

| Сервіс | Scope |
|---|---|
| Gmail | `gmail.modify` |
| Google Docs | `documents` |
| Google Sheets | `spreadsheets` |
| Google Drive | `drive` |
| Google Calendar | `calendar` |
| Google Meet | `calendar` (посилання Meet створюються через Calendar API) |

## Сценарії використання

<AccordionGroup>
  <Accordion title="Автоматичні follow-up листи">
    Агент кваліфікує ліда через чат → використовує `send_email` для надсилання follow-up з цінами → фіксує взаємодію в `Google Sheets` через `append_rows`.
  </Accordion>
  <Accordion title="Запис на зустріч">
    Клієнт просить запланувати зустріч → агент використовує `check_availability` для пошуку вільних слотів → `create_event` для бронювання → `create_meeting` для генерації посилання Google Meet → надсилає підтвердження через чат або електронну пошту.
  </Accordion>
  <Accordion title="Генерація документів">
    Агент збирає інформацію від клієнта → використовує `create_document` для створення комерційної пропозиції або звіту → `share_file` для надання клієнту доступу.
  </Accordion>
  <Accordion title="Відстеження лідів у стилі CRM">
    Кожна розмова, що завершується лідом → агент використовує `append_rows` для додавання даних ліда в Google Sheet → `send_email` для сповіщення команди продажів.
  </Accordion>
</AccordionGroup>

## Оновлення токенів

Access token Google закінчуються через 1 годину. Revol обробляє це автоматично:

1. Перед кожним API-викликом Revol перевіряє, чи не прострочений access token
2. Якщо прострочений, використовує збережений refresh token для отримання нового access token
3. Новий токен шифрується та зберігається назад у базу даних
4. API-виклик виконується зі свіжим токеном

Якщо refresh token стає недійсним (користувач скасував доступ), інтеграція позначається як відключена, і користувачу пропонується повторно авторизуватися.
