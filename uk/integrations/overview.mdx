## Що таке інтеграції?

Інтеграції з'єднують ваших AI-агентів із зовнішніми сервісами — щоб вони могли відповідати на телефонні дзвінки, писати у месенджерах, надсилати електронні листи, створювати документи та отримувати дані аналітики. Кожна інтеграція надає вашому агенту набір **інструментів**, які він може викликати під час розмов.

## Архітектура

Кожна інтеграція працює за однаковим принципом:

<Steps>
  <Step title="Підключення">
    Додайте інтеграцію через майстер. Залежно від провайдера, ви або вводите API-облікові дані (VoIP, Telegram), або проходите OAuth (Google, Meta).
  </Step>
  <Step title="Перевірка">
    Revol перевіряє ваші облікові дані через API провайдера та отримує доступні ресурси (номери телефонів, інформацію про бота, підключені акаунти).
  </Step>
  <Step title="Створення інструментів">
    Після перевірки Revol створює **інструменти агента** відповідно до типу інтеграції. Наприклад, підключення Gmail дає вашим агентам інструменти `send_email`, `read_inbox`, `search_emails` та інші.
  </Step>
  <Step title="Призначення агенту">
    У налаштуваннях агента увімкніть інструменти, які він має використовувати. Тепер агент може викликати ці інструменти під час розмов — автоматично через вузли workflow або через LLM function calling.
  </Step>
</Steps>

## Методи автентифікації

| Метод | Використовується | Як працює |
|---|---|---|
| **API Key / Token** | Twilio, Ringostat, Binotel, Telegram | Введіть облікові дані в майстрі. Зберігаються зашифрованими в базі даних. |
| **Google OAuth** | Gmail, Docs, Sheets, Drive, Calendar, Meet, Google Ads | Користувач натискає "Підключити", авторизується на екрані згоди Google, Revol зберігає refresh token. |
| **Meta OAuth** | WhatsApp, Instagram, Facebook Messenger, Facebook Ads | Користувач натискає "Підключити", авторизується через Meta, Revol обмінює на довготривалий токен (60 днів). |
| **Webhook URL** | Custom Webhook | Автентифікація не потрібна — ви вказуєте цільову URL-адресу, і Revol надсилає дані на неї. |

## Безпека Webhook

Усі вхідні webhook-ендпоінти (VoIP-дзвінки, повідомлення месенджерів) захищені:

- **Перевірка підпису** — криптографічна верифікація автентичності запиту для кожного провайдера
- **Ідемпотентність** — виявлення дублікатів webhook для запобігання подвійній обробці
- **Обмеження частоти** — 120 запитів на хвилину на канал
- **Білий список IP** — де підтримується провайдером (наприклад, Binotel)

## Дорожня карта реалізації

<Steps>
  <Step title="Phase 1 — Telegram">
    Інтеграція з Telegram Bot API. OAuth не потрібен — лише токен бота від BotFather. Включає обробку webhook для вхідних повідомлень та 6 інструментів агента для надсилання, редагування та керування повідомленнями.
  </Step>
  <Step title="Phase 2 — Google Workspace">
    Шість інтеграцій через єдиний SDK `google/apiclient`. Спільний потік OAuth з оновленням токенів. Охоплює Gmail, Google Docs, Sheets, Drive, Calendar та Meet — приблизно 38 інструментів агента загалом.
  </Step>
  <Step title="Phase 3 — Meta Messengers">
    WhatsApp Business, Instagram DM та Facebook Messenger через Meta Graph API. Спільний потік Meta OAuth з обміном на довготривалий токен. Обробка webhook для вхідних повідомлень на всіх трьох платформах — приблизно 16 інструментів агента.
  </Step>
  <Step title="Phase 4 — Webhook та реклама">
    Custom Webhook для інтеграції з CRM/зовнішніми системами. Google Ads та Facebook Ads для аналітики кампаній. Приблизно 5 інструментів агента.
  </Step>
</Steps>
