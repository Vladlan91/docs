## Підтримувані провайдери

| Провайдер | Вхідні дзвінки | Вихідні дзвінки | SMS | Автоналаштування Webhook |
|---|---|---|---|---|
| **Twilio** | Так | Так | Так | Так (автоматично) |
| **Ringostat** | Так | Так | Ні | Вручну |
| **Binotel** | Так | Так | Ні | Вручну |

## Налаштування

<Tabs>
  <Tab title="Twilio">
    ### Облікові дані

    Вам потрібні три значення з [Twilio Console](https://console.twilio.com/):

    | Поле | Де знайти |
    |---|---|
    | **Account SID** | Dashboard → Account Info |
    | **Auth Token** | Dashboard → Account Info |

    ### Кроки

    <Steps>
      <Step title="Додайте інтеграцію">
        Перейдіть до **Інтеграції** → **Додати інтеграцію** → оберіть **Twilio** у категорії Телефонія.
      </Step>
      <Step title="Введіть облікові дані">
        Оберіть компанію, дайте інтеграції назву (наприклад, "Основна лінія"), та введіть Account SID і Auth Token.
      </Step>
      <Step title="Перевірте та оберіть номер">
        Натисніть **Перевірити з'єднання**. Revol отримає список ваших доступних номерів телефонів. Оберіть номер для використання.
      </Step>
      <Step title="Збережіть">
        Revol автоматично налаштовує webhook URL на обраному номері Twilio. Вхідні дзвінки та SMS спрямовуються до вашого агента.
      </Step>
    </Steps>

    <Note>
      Webhook URL для Twilio налаштовуються автоматично. Вам не потрібно нічого змінювати в Twilio Console.
    </Note>
  </Tab>
  <Tab title="Ringostat">
    ### Облікові дані

    | Поле | Де знайти |
    |---|---|
    | **API Key (Auth-key)** | Ringostat Dashboard → API Settings |
    | **Project ID** | Ringostat Dashboard → Project Settings |

    ### Кроки

    <Steps>
      <Step title="Додайте інтеграцію">
        Перейдіть до **Інтеграції** → **Додати інтеграцію** → оберіть **Ringostat**.
      </Step>
      <Step title="Введіть облікові дані">
        Введіть ваш API Key та Project ID.
      </Step>
      <Step title="Перевірте">
        Revol отримає ваші SIP-акаунти для підтвердження з'єднання.
      </Step>
      <Step title="Налаштуйте Webhook">
        У панелі Ringostat встановіть webhook URL:
        ```
        https://app.revol.com/api/webhooks/ringostat/{integration_id}
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="Binotel">
    ### Облікові дані

    | Поле | Де знайти |
    |---|---|
    | **API Key** | Binotel Dashboard → Settings → API |
    | **API Secret** | Binotel Dashboard → Settings → API |

    ### Кроки

    <Steps>
      <Step title="Додайте інтеграцію">
        Перейдіть до **Інтеграції** → **Додати інтеграцію** → оберіть **Binotel**.
      </Step>
      <Step title="Введіть облікові дані">
        Введіть ваш API Key та API Secret.
      </Step>
      <Step title="Перевірте">
        Revol отримає ваші внутрішні номери телефонів для підтвердження з'єднання.
      </Step>
      <Step title="Налаштуйте Webhook">
        У панелі Binotel встановіть webhook URL:
        ```
        https://app.revol.com/api/webhooks/binotel/{integration_id}
        ```
      </Step>
    </Steps>

    <Warning>
      Webhook Binotel перевіряються за білим списком IP. У production приймаються лише запити з серверів Binotel.
    </Warning>
  </Tab>
</Tabs>

## Інструменти агента

Після підключення VoIP-провайдера наступні інструменти автоматично створюються для ваших агентів:

| Інструмент | Twilio | Ringostat | Binotel | Опис |
|---|---|---|---|---|
| **make_call** | Так | Так | Так | Ініціювати вихідний дзвінок |
| **send_sms** | Так | — | — | Надіслати SMS-повідомлення |
| **get_call_history** | Так | Так | Так | Отримати журнал останніх дзвінків |

## Потік голосового дзвінка

Коли надходить вхідний дзвінок, Revol обробляє його через workflow призначеного агента:

<Steps>
  <Step title="Вхідний дзвінок">
    VoIP-провайдер надсилає webhook-сповіщення до Revol з інформацією про абонента.
  </Step>
  <Step title="Розпізнавання мовлення">
    Мовлення абонента передається потоком та транскрибується в реальному часі за допомогою Whisper або Google STT.
  </Step>
  <Step title="AI-обробка">
    Транскрибований текст надсилається до workflow агента. Агент обробляє його через LLM-вузли, умови та виклики інструментів.
  </Step>
  <Step title="Синтез мовлення">
    Текстова відповідь агента конвертується в аудіо за допомогою OpenAI TTS з налаштовуваним голосом і швидкістю.
  </Step>
  <Step title="Відповідь">
    Аудіо передається потоком назад абоненту. Агент також може виконувати дії під час дзвінка — переключити на оператора, знайти дані або завершити дзвінок.
  </Step>
</Steps>

## Безпека Webhook

| Провайдер | Метод перевірки |
|---|---|
| Twilio | Верифікація підпису HMAC-SHA1 |
| Ringostat | Структурна перевірка параметрів |
| Binotel | Білий список IP (лише production) |

Усі VoIP webhook обмежені до **120 запитів на хвилину** на канал.
