## Компанії

**Компанія** — це центральна сутність у Revol. Все — продукти, файли, агенти, інтеграції та аналітика — належить компанії.

Кожна компанія має:

- **Продукти та послуги** — структурований каталог з описами, цінами, фото та параметрами
- **Файловий менеджер** — документи, посилання, текст, відео та фото для баз знань
- **AI Агенти** — один або кілька інтелектуальних асистентів, навчених на даних компанії
- **Інтеграції** — підключені канали (widget, VoIP, месенджери, реклама, CRM)
- **Аналітика** — відстеження відвідувачів, дані сесій та історія розмов

<Note>
  Один акаунт Revol може керувати кількома компаніями з повністю ізольованими даними — ідеально для агентств або мультибрендових бізнесів.
</Note>

## Продукти та послуги

**Продукти** — це структуровані моделі даних, що описують те, що пропонує ваш бізнес. На відміну від звичайних текстових документів, продукти надають AI агентам точну, структуровану інформацію для пошуку.

Кожен продукт включає:

| Поле | Призначення |
|------|-------------|
| **Назва** | Назва продукту або послуги |
| **Опис** | Детальний текст, який агент використовує для відповідей |
| **Ціна** | Інформація про ціну |
| **URL** | Посилання на сторінку продукту на вашому сайті |
| **Наявність** | Чи доступний продукт наразі |
| **Фото** | Зображення продукту, які агент може надсилати |
| **Параметри** | Структуровані пари ключ-значення (розмір, колір, тривалість тощо) |

**Параметри** — найпотужніша частина — структуровані характеристики, які дозволяють агентам давати точні відповіді замість інтерпретації неструктурованого тексту. Дивіться [Швидкий старт → Параметри продуктів](/quickstart#product-parameters) для прикладів.

## База знань

**База знань** — це контент, на якому навчаються ваші AI агенти. Вона використовує **RAG (Retrieval-Augmented Generation)** — документи розбиваються на фрагменти, перетворюються на векторні embeddings і семантично шукаються під час розмов.

### Типи контенту

| Тип | Формати | Як це працює |
|-----|---------|--------------|
| **Документи** | PDF, DOCX, TXT | Текст витягується та розбивається на фрагменти |
| **Посилання** | Будь-який URL | Вміст веб-сторінки завантажується та індексується |
| **Текст** | Довільний текст | Додається безпосередньо як фрагменти знань |
| **Відео** | MP4, WebM, MOV | Аудіо транскрибується в текст через STT |
| **Фото** | JPG, PNG | Текст витягується через OCR |

### Два рівні знань

- **Файли на рівні компанії** — доступні всім агентам компанії
- **Файли на рівні продукту** — прикріплені до конкретного продукту, використовуються коли агент обговорює цей продукт

## AI Агенти

**AI Агент** — це інтелектуальний асистент, який обробляє взаємодію з клієнтами. Кожен агент налаштовується незалежно зі своєю особистістю, знаннями та поведінкою.

### Компоненти агента

<AccordionGroup>
  <Accordion title="Системний промпт">
    Основний набір інструкцій — визначає роль агента, тон, мову та обмеження. Чим конкретніший ваш промпт, тим краще працює агент.
  </Accordion>
  <Accordion title="База знань (RAG)">
    Оберіть, з яких продуктів та файлів агент має навчатися. Після вибору натисніть **Тренувати**, щоб згенерувати векторні embeddings для семантичного пошуку.
  </Accordion>
  <Accordion title="Workflow">
    Візуальний редактор на основі вузлів для логіки розмови. Кожен агент отримує [стандартний workflow](/quickstart#default-workflow), який покриває більшість випадків використання з коробки.
  </Accordion>
  <Accordion title="Інструменти">
    Функції, які агент може викликати під час розмов — пошук продуктів, перевірка наявності, отримання фото, завантаження документів, отримання інформації про компанію.
  </Accordion>
  <Accordion title="Widget">
    Налаштування зовнішнього вигляду чат-віджета — кольори, підписи, розташування, привітальні повідомлення, умови використання.
  </Accordion>
  <Accordion title="Канали">
    Підключення агента до каналів комунікації — віджет на сайті, Telegram, WhatsApp, Instagram, Facebook Messenger, VoIP.
  </Accordion>
</AccordionGroup>

### LLM Провайдери

Кожен агент (або окремий вузол workflow) може використовувати різну AI модель:

| Провайдер | Стандартні моделі | Преміум моделі |
|-----------|-------------------|----------------|
| **OpenAI** | GPT-4o-mini | GPT-4o |
| **Anthropic** | Claude 3.5 Haiku | Claude 3.5 Sonnet |
| **Google** | Gemini 2.0 Flash | Gemini 2.5 Pro |
| **Groq** | Llama 3.1 8B | Llama 3.3 70B |

<Info>
  Стандартні моделі швидші та споживають стандартні токени. Преміум моделі пропонують вищу якість та споживають преміум токени.
</Info>

## Workflows

**Workflow** — це візуальний редактор на основі вузлів для визначення того, як ваш агент обробляє розмови. Вузли з'єднані зв'язками, що визначають потік даних.

### Типи вузлів

| Вузол | Опис |
|-------|------|
| **Start** | Точка входу — отримує повідомлення користувача |
| **LLM** | Надсилає контекст до AI моделі та отримує відповідь |
| **Condition** | Розгалужує потік на основі правил (ключові слова, намір, змінні) |
| **Tool Call** | Виконує зовнішню функцію (пошук продуктів, перевірка наявності) |
| **Voice Input (STT)** | Перетворює мовлення в текст для голосових взаємодій |
| **Voice Output (TTS)** | Перетворює текстову відповідь у мовлення |
| **Response Formatter** | Об'єднує виходи з кількох вузлів в одну відповідь |

### Стандартний Workflow

Кожен новий агент має попередньо побудований workflow:

```
Start → Product Agent  ─┐
     ├→ Media Agent    ─┤→ Response Formatter
     └→ Company Agent  ─┘
```

Три спеціалізовані агенти працюють **паралельно** — кожен обробляє свою область (продукти, медіа, інформація про компанію), а Formatter об'єднує їхні виходи. Це покриває більшість випадків використання без будь-якого налаштування.

Для складних сценаріїв — умовне розгалуження, власні виклики інструментів, багатокрокові потоки — використовуйте [Редактор Workflow](/agents/workflow).

## Розмови

**Розмова** — це чат-сесія між відвідувачем та AI агентом. Усі розмови зберігаються та доступні зі сторінки **Розмови**.

### Канали

Розмови можуть відбуватися через:

- **Віджет на сайті** — вбудований чат на вашому сайті
- **VoIP** — телефонні дзвінки через Twilio, Ringostat або Binotel
- **Месенджери** — Telegram, WhatsApp Business, Instagram DM, Facebook Messenger
- **API** — програмний доступ

### Можливості

- **Єдина скринька** — всі розмови з усіх агентів та каналів в одному місці
- **Фільтри** — за агентом, статусом, датою, каналом
- **Транскрипти** — повна історія повідомлень з мітками часу
- **Відтворення аудіо** — прослуховування записів голосових дзвінків
- **Рейтинги** — оцінки задоволеності клієнтів за кожну розмову

## Трекінг

**Tracker** вбудований у той самий скрипт віджета та автоматично відстежує активність відвідувачів на вашому сайті.

### Що відстежується

| Дані | Опис |
|------|------|
| **Сесії** | Сесії перегляду відвідувачів — пристрій, браузер, ОС, локація, джерело переходу |
| **Перегляди сторінок** | Окремі відвідування сторінок в межах сесії |
| **Події** | Користувацькі події — кліки, відправки форм, телефонні дзвінки |
| **Кампанії** | Групування на основі UTM (source, medium, campaign, term, content) |

### Автоматизація

- **Тригери** — автоматичні дії на основі поведінки відвідувачів (час на сторінці, глибина прокрутки, збіг UTM, тип пристрою). Можуть показувати оверлеї чату або користувацькі повідомлення.
- **Підміна номерів** — динамічна заміна телефонних номерів для відстеження та атрибуції дзвінків.

## Інтеграції

**Інтеграції** підключають ваших AI агентів до зовнішніх сервісів та каналів комунікації.

| Категорія | Провайдери | Що це робить |
|-----------|------------|--------------|
| **VoIP** | Twilio, Ringostat, Binotel | Телефонні дзвінки — агент відповідає на вхідні дзвінки |
| **Месенджери** | Telegram, WhatsApp, Instagram, Facebook Messenger | Чат через месенджери |
| **Google Workspace** | Gmail, Docs, Sheets, Drive, Calendar, Meet | Читання та взаємодія з сервісами Google |
| **Реклама** | Google Ads, Facebook Ads | Перегляд ефективності кампаній та управління рекламою |
| **Webhook** | Будь-який HTTP endpoint | Надсилання даних лідів, подій або сповіщень до CRM чи інших систем |

Кожна інтеграція налаштовується для кожної компанії окремо та може бути підключена до одного або кількох агентів.

### Що вважається токеном

Кожна AI взаємодія має дві частини:

- **Вхідні токени** — ваше повідомлення + системний промпт + отримані фрагменти бази знань + історія розмови
- **Вихідні токени** — відповідь агента

Різні моделі споживають різні типи токенів:

| Тип токенів | Моделі | Плани |
|-------------|--------|-------|
| **Стандартні** | GPT-4o-mini, Claude 3.5 Haiku, Gemini 2.0 Flash, Llama 3.1 8B | Усі плани |
| **Преміум** | GPT-4o, Claude 3.5 Sonnet, Gemini 2.5 Pro, Llama 3.3 70B | Premium+ |

### Що відбувається при досягненні ліміту

| Ресурс | Поведінка |
|--------|-----------|
| Стандартні / Преміум токени | Агент перестає відповідати до наступного білінгового циклу або покупки пакету |
| Сховище | Неможливо завантажити нові файли |
| Голос | Голосові функції вимкнено |
| Розмови | Нові розмови заблоковано |

Revol надсилає сповіщення при **80%** (попередження) та **90%** (критичне) використання, щоб ви могли діяти до досягнення ліміту.
