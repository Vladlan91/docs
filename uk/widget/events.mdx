## Огляд

Вкладка Events дозволяє визначати кастомні події для відстеження конкретних дій відвідувачів на вашому сайті. Події можуть відстежувати кліки, відправлення форм, досягнення порогів прокрутки, видимість елементів та інше.<Frame>

## Створення події

Натисніть **Create Event**, щоб визначити нову подію:

| Поле | Опис |
|---|---|
| **Name** | Назва події для ідентифікації |
| **Type** | Тип події: Click, Form Submit, Scroll, Visibility, Page View або Custom |
| **Selector** | CSS-селектор цільового елемента (для типів Click, Form, Visibility) |
| **Page URL** | Необов'язково — обмежити подію конкретною сторінкою |
| **Conversion Goal** | Перемикач для позначення події як конверсії |

## Типи подій

| Тип | Опис |
|---|---|
| **Click** | Спрацьовує при кліку на вказаний елемент |
| **Form Submit** | Спрацьовує при відправленні форми, що відповідає селектору |
| **Scroll** | Спрацьовує при прокрутці за визначений поріг відсотків |
| **Visibility** | Спрацьовує коли вказаний елемент стає видимим у viewport |
| **Page View** | Спрацьовує при завантаженні конкретної сторінки |
| **Custom** | Спрацьовує через JavaScript API: `revolTracker.event('name')` |

## Конверсійні цілі

Будь-яка подія може бути позначена як **Conversion Goal** перемикачем. Конверсійні події:

- Враховуються у KPI **Conversions** на Dashboard
- Використовуються для розрахунку **Conversion Rate** у таблиці ефективності кампаній
- Виділяються у детальному перегляді сесії

## Список подій

Таблиця подій показує всі визначені події з:

| Колонка | Опис |
|---|---|
| **Name** | Назва події |
| **Type** | Тип події |
| **Fires** | Загальна кількість спрацювань |
| **Conversion** | Чи є подія конверсійною ціллю |
| **Actions** | Редагувати або видалити подію |
