## Огляд

Сторінка Sessions надає детальний перегляд усіх сесій відвідувачів у рамках кампанії. Натисніть на будь-який рядок кампанії у вкладці Campaigns, щоб відкрити **повноекранний переглядач сесій** — потужний інструмент для розуміння того, як відвідувачі переміщуються вашим сайтом, які сторінки переглядають і чи починають розмови з вашим AI-агентом.

## Переглядач сесій кампанії

Переглядач сесій відкривається як повноекранне модальне вікно з назвою кампанії, бейджем статусу та ID у заголовку. Заголовок також надає швидкі дії:

| Дія | Опис |
|---|---|
| **Embed** | Отримати код для вбудовування трекінгового посилання кампанії |
| **Preview** | Попередній перегляд лендінгу кампанії |
| **Settings** | Швидкий доступ до source, medium, ID кампанії та дій редагування/видалення |

Переглядач має дві основні вкладки: **Live** та **Sessions**.

---

## Вкладка Live

Вкладка Live показує **активність відвідувачів у реальному часі** для кампанії. Коли відвідувачі активно переглядають ваш сайт через трекінгове посилання цієї кампанії, їхні переходи між сторінками з'являються тут у реальному часі.

### Live Canvas

Ліва частина відображає граф потоку сторінок у реальному часі. Коли відвідувачі переміщуються між сторінками, граф оновлюється з анімованими з'єднаннями, що показують шлях від однієї сторінки до іншої.

**Елементи керування:**

| Кнопка | Опис |
|---|---|
| **Refresh** | Перезавантажити дані в реальному часі |
| **Zoom In / Out** | Змінити масштаб графа |
| **Reset Zoom** | Повернутися до масштабу за замовчуванням |

Коли немає активних відвідувачів, canvas показує порожній стан: *«No live sessions — When visitors navigate this campaign, their page transitions will appear here in real time.»*

### Панель активних відвідувачів

Права частина показує список **активних відвідувачів** з індикатором онлайн (пульсуюча зелена точка) та кількістю. Кожна картка активного відвідувача показує тип пристрою, браузер та поточну сторінку.

---

## Вкладка Sessions

Вкладка Sessions показує **всі історичні сесії**, атрибутовані до цієї кампанії. Тут ви аналізуєте минулу поведінку відвідувачів у деталях.

### Граф потоку сторінок

Ліва частина відображає візуальний **граф потоку сторінок** для вибраної сесії — вертикальний ланцюг вузлів сторінок, що показує подорож відвідувача вашим сайтом.

#### Фільтри-піли

У верхній частині графа два ряди інтерактивних фільтрів-пілів допомагають звузити вибірку сесій:

| Фільтр | Опис |
|---|---|
| **Devices** | Фільтр за Desktop, Tablet або Mobile. Натисніть піл, щоб показати лише сесії з цього типу пристрою. |
| **Traffic Sources** | Фільтр за джерелом — Google, Facebook, Instagram, TikTok, Telegram, Direct та 20+ інших джерел. Кожне джерело показує свій логотип для швидкої ідентифікації. |

Активні фільтри підсвічуються. Натисніть піл, щоб увімкнути/вимкнути його. Список сесій та граф оновлюються відповідно.

#### Вузли сторінок

Кожна сторінка у подорожі відвідувача показується як **вузол сторінки** з:

| Елемент | Опис |
|---|---|
| **Номер** | Порядковий номер сторінки у візиті (1, 2, 3...) |
| **Заголовок сторінки** | HTML-заголовок сторінки |
| **URL сторінки** | Шлях URL (наприклад, `/portfolio`, `/terrazzo`) |
| **Бейдж часу** | Час, проведений на сторінці (наприклад, «54s», «1m 47s») — зліва |
| **Бейдж прокрутки** | Відсоток глибини прокрутки (наприклад, «72%», «60%») — справа |

Вузли з'єднані **SVG-лініями**, що показують потік навігації. Натисніть будь-який вузол сторінки, щоб переглянути його повні деталі у правій панелі.

### Права панель

Права панель має три під-вкладки: **Sessions**, **Details** та **Conversations**.

#### Під-вкладка Sessions

Прокручуваний список усіх сесій кампанії. Кожна картка сесії показує:

| Елемент | Опис |
|---|---|
| **Іконка джерела** | Логотип джерела трафіку (Google, Facebook, Telegram, TikTok, Direct тощо) |
| **Іконка пристрою** | Desktop, планшет або мобільний |
| **Токен відвідувача** | Унікальний анонімний ідентифікатор (наприклад, `rvl_Bf0G`) |
| **Бейдж Bounce** | Червона мітка «Bounce», якщо відвідувач переглянув лише одну сторінку |
| **Час тому** | Відносний час з моменту сесії (наприклад, «2d ago», «15d ago») |
| **Кількість сторінок** | Кількість переглянутих сторінок |
| **Тривалість** | Загальна тривалість сесії (наприклад, «6m 54s») |
| **Браузер** | Назва браузера (Chrome, Safari, Firefox, Edge, Samsung Internet) |
| **Локація** | Код країни та місто (наприклад, «UA, Київ», «DE, Hamburg») |

Натисніть на картку сесії, щоб завантажити її граф потоку сторінок зліва. Активна сесія підсвічується.

У заголовку показано загальну кількість сесій та кнопку **Refresh** для перезавантаження списку.

#### Під-вкладка Details

Коли ви натискаєте на **вузол сторінки** у графі, під-вкладка Details показує повну інформацію про цей конкретний перегляд:

**Інформація про сторінку:**

| Поле | Опис |
|---|---|
| **Title** | Повний заголовок сторінки |
| **URL** | Повний URL сторінки (з посиланням для відкриття у новій вкладці) |
| **Referrer** | URL попередньої сторінки, з якої прийшов відвідувач |

**Залучення:**

| Поле | Опис |
|---|---|
| **Time on page** | Скільки часу відвідувач провів на сторінці |
| **Scroll depth** | Наскільки далеко відвідувач прокрутив сторінку (відсоток) |
| **Visited at** | Точна дата та час перегляду |

Якщо вузол сторінки не вибрано, панель показує: *«Select a page node to view details.»*

#### Під-вкладка Conversations

Якщо відвідувач розпочав **чат-розмову** з вашим AI-агентом під час сесії, тут з'являється повний транскрипт розмови — включаючи повідомлення відвідувача та відповіді агента.

Якщо до сесії не прив'язана розмова, панель показує: *«No conversation linked to this session.»*

<Tip>
  Під-вкладка Conversations дозволяє бачити точний контекст розмови — які сторінки відвідувач переглядав перед початком чату, скільки часу провів і що обговорював з AI-агентом. Це безцінно для розуміння намірів відвідувачів та покращення відповідей агента.
</Tip>

## Індикатори сесій

Сесії використовують візуальні індикатори для швидкої передачі ключової інформації:

| Індикатор | Значення |
|---|---|
| **Логотип джерела** | Звідки прийшов відвідувач (Google, Facebook, Instagram, TikTok, Telegram, Direct тощо) |
| **Іконка пристрою** | Монітор, планшет або мобільний телефон |
| **Бейдж «Bounce»** | Відвідувач переглянув лише 1 сторінку та пішов |
| **Повторний відвідувач** | Той самий `rvl_`-токен з'являється у кількох сесіях |
| **Бейджі часу на вузлах** | Час, проведений на кожній окремій сторінці |
| **Бейджі прокрутки на вузлах** | Наскільки далеко відвідувач прокрутив кожну сторінку |
