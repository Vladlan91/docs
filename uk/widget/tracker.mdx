## Огляд

Revol Tracker відстежує активність відвідувачів на вашому сайті. Він збирає дані сесій, перегляди сторінок, кастомні події та атрибуцію кампаній — все через той самий скрипт, що забезпечує роботу чат-віджета. Додатковий код не потрібен.

Dashboard надає огляд трафіку та поведінки відвідувачів вашого сайту в реальному часі. Дані оновлюються автоматично кожні **60 секунд**.

Використовуйте фільтри для звуження даних дашборду:

**Швидкий вибір періоду:**

| Період | Опис |
|---|---|
| **24h** | Останні 24 години |
| **7d** | Останні 7 днів (за замовчуванням) |
| **30d** | Останні 30 днів |
| **90d** | Останні 90 днів |

**Розширені фільтри** (dropdown):

| Фільтр | Опис |
|---|---|
| **Device** | Desktop, Mobile або Tablet |
| **Country** | Фільтр за країною (2-літерний ISO код) |
| **Source** | UTM source (напр., google, facebook) |
| **Page URL** | Фільтр за конкретною сторінкою |
| **Browser** | Chrome, Firefox, Safari тощо |
| **OS** | Windows, macOS, iOS, Android тощо |
| **Bounce** | Показати лише відмовні або не відмовні сесії |
| **Campaign** | Вибір зі списку ваших кампаній |
| **Medium** | cpc, organic, social, email, referral |
| **Date From** | Кастомна початкова дата |
| **Date To** | Кастомна кінцева дата |

<Tip>
  Без фільтрів дашборд використовує попередньо агреговані дані для швидкого завантаження. При застосуванні фільтрів переключається на прямі запити для точних результатів — це може зайняти трохи більше часу.
</Tip>

### Sessions (Сесії)

Загальна кількість сесій відвідувачів за обраний період. Сесія починається, коли відвідувач завантажує ваш сайт, і завершується після 30 хвилин неактивності або при виході.

**Розрахунок:** `COUNT(*)` з усіх сесій за період.

### Unique Visitors (Унікальні відвідувачі)

Кількість окремих відвідувачів, ідентифікованих за унікальним токеном браузера. Одна людина, що відвідала ваш сайт 5 разів = 1 унікальний відвідувач, але 5 сесій.

**Розрахунок:** `COUNT(DISTINCT guest_token)` з сесій.

### Bounce Rate (Показник відмов)

Відсоток сесій, де відвідувач переглянув лише одну сторінку та покинув сайт без подальшої взаємодії. Нижчий показник відмов зазвичай означає кращу залученість.

**Розрахунок:** `(сесії з 1 переглядом / всі сесії) × 100%`

<Note>
  Стрілка тренду для показника відмов **інвертована** — зменшення (зелена стрілка) означає покращення, а збільшення (червона стрілка) означає, що більше відвідувачів йдуть після однієї сторінки.
</Note>

### Avg Duration (Середня тривалість)

Середній час, який відвідувачі проводять на вашому сайті за сесію, у секундах або хвилинах.

**Розрахунок:** `AVG(duration_seconds)` по всіх сесіях.

### Pages / Visit (Сторінок за візит)

Середня кількість переглянутих сторінок за сесію. Вищі значення вказують, що відвідувачі вивчають більше вашого контенту.

**Розрахунок:** `AVG(pages_visited)` по всіх сесіях.

### Conversions (Конверсії)

Кількість унікальних сесій, які запустили принаймні одну подію, позначену як **Conversion Goal**. Ви самі визначаєте, які події вважаються конверсіями, у вкладці Events.

**Розрахунок:** `COUNT(DISTINCT session_id)` де сесія містить принаймні одну подію з `is_conversion = true`.

<Tip>
  Щоб налаштувати конверсії, перейдіть у вкладку **Events**, створіть або відредагуйте подію та увімкніть перемикач **"Conversion Goal"**. Будь-який тип події може бути конверсією — відправлення форм, кліки на кнопки, перегляди сторінок або кастомні події.
</Tip>

## Sessions Over Time (Сесії за часом)

Графік-область, що показує тренди трафіку за обраний період. Дві накладені лінії:

- **Sessions** — загальна кількість сесій за день
- **Visitors** — унікальні відвідувачі за день

Використовуйте для виявлення піків трафіку, трендів та розриву між загальними візитами та унікальними відвідувачами (коефіцієнт повторних візитів).

## New vs Returning (Нові vs Повторні)

Кільцева діаграма, що показує розподіл між:

- **New visitors** — відвідувачі вперше (лише 1 сесія за весь час)
- **Returning visitors** — відвідувачі, які вже були на вашому сайті (2+ сесій)

Здоровий баланс залежить від бізнесу: контентні сайти прагнуть більше повторних відвідувачів, тоді як рекламні лендинги очікують переважно нових.

## Session Duration (Тривалість сесій)

Гістограма, що показує, скільки часу відвідувачі проводять на вашому сайті, згрупована у 6 бакетів:

| Бакет | Опис |
|---|---|
| **0–10с** | Дуже короткі візити (ймовірно відмови) |
| **10–30с** | Швидкі перегляди |
| **30–60с** | Короткі візити |
| **1–3 хв** | Помірна залученість |
| **3–10 хв** | Хороша залученість |
| **10+ хв** | Глибока залученість |

## Scroll Depth (Глибина прокрутки)

Гістограма, що показує, наскільки далеко відвідувачі прокручують ваші сторінки, у 4 бакетах:

| Бакет | Опис |
|---|---|
| **0–25%** | Побачили лише верх сторінки |
| **25–50%** | Прокрутили до середини |
| **50–75%** | Прочитали більшу частину |
| **75–100%** | Дійшли до низу |

Використовуйте, щоб зрозуміти, чи читають відвідувачі ваш контент повністю, чи залишають сторінку рано.
