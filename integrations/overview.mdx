## What Are Integrations?

Integrations connect your AI agents to external services — so they can answer phone calls, reply in messengers, send emails, create documents, and pull analytics data. Each integration gives your agent a set of **tools** it can call during conversations.

## Architecture

Every integration follows the same pattern:

<Steps>
  <Step title="Connect">
    Add an integration from the wizard. Depending on the provider, you either enter API credentials (VoIP, Telegram) or go through OAuth (Google, Meta).
  </Step>
  <Step title="Verify">
    Revol tests your credentials against the provider's API and fetches available resources (phone numbers, bot info, connected accounts).
  </Step>
  <Step title="Provision Tools">
    After verification, Revol creates **agent tools** based on the integration type. For example, connecting Gmail gives your agents `send_email`, `read_inbox`, `search_emails`, and other tools.
  </Step>
  <Step title="Assign to Agent">
    In the agent settings, enable the tools you want the agent to use. The agent can now call these tools during conversations — either automatically through workflow nodes or via LLM function calling.
  </Step>
</Steps>

## Authentication Methods

| Method | Used by | How it works |
|---|---|---|
| **API Key / Token** | Twilio, Ringostat, Binotel, Telegram | Enter credentials in the wizard. Stored encrypted in database. |
| **Google OAuth** | Gmail, Docs, Sheets, Drive, Calendar, Meet, Google Ads | User clicks "Connect", authorizes in Google consent screen, Revol stores refresh token. |
| **Meta OAuth** | WhatsApp, Instagram, Facebook Messenger, Facebook Ads | User clicks "Connect", authorizes via Meta, Revol exchanges for long-lived token (60 days). |
| **Webhook URL** | Custom Webhook | No auth needed — you provide the target URL and Revol sends data to it. |

## Webhook Security

All inbound webhook endpoints (VoIP calls, messenger messages) are protected with:

- **Signature validation** — cryptographic verification of request authenticity per provider
- **Idempotency** — duplicate webhook detection to prevent double-processing
- **Rate limiting** — 120 requests per minute per channel
- **IP allowlisting** — where supported by provider (e.g., Binotel)

## Implementation Roadmap

<Steps>
  <Step title="Phase 1 — Telegram">
    Telegram Bot API integration. No OAuth required — just a bot token from BotFather. Includes webhook handling for incoming messages and 6 agent tools for sending, editing, and managing messages.
  </Step>
  <Step title="Phase 2 — Google Workspace">
    Six integrations via a single `google/apiclient` SDK. Shared OAuth flow with token refresh. Covers Gmail, Google Docs, Sheets, Drive, Calendar, and Meet — approximately 38 agent tools total.
  </Step>
  <Step title="Phase 3 — Meta Messengers">
    WhatsApp Business, Instagram DM, and Facebook Messenger via Meta Graph API. Shared Meta OAuth flow with long-lived token exchange. Webhook handling for incoming messages across all three platforms — approximately 16 agent tools.
  </Step>
  <Step title="Phase 4 — Webhook & Ads">
    Custom Webhook for CRM/external system integration. Google Ads and Facebook Ads for campaign analytics. Approximately 5 agent tools.
  </Step>
</Steps>
