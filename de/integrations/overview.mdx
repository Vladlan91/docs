## Was sind Integrationen?

Integrationen verbinden Ihre KI-Agenten mit externen Diensten — damit sie Telefonanrufe beantworten, in Messengern antworten, E-Mails senden, Dokumente erstellen und Analysedaten abrufen koennen. Jede Integration gibt Ihrem Agenten eine Reihe von **Tools**, die er waehrend Konversationen aufrufen kann.

## Funktionsweise

Jede Integration folgt demselben Muster:

<Steps>
  <Step title="Verbinden">
    Fuegen Sie eine Integration von der Integrationen-Seite hinzu. Je nach Anbieter geben Sie entweder API-Zugangsdaten ein (VoIP, Telegram) oder durchlaufen OAuth (Google, Meta).
  </Step>
  <Step title="Verifizieren">
    Revol testet Ihre Zugangsdaten gegen die API des Anbieters und ruft verfuegbare Ressourcen ab (Telefonnummern, Bot-Info, verbundene Konten).
  </Step>
  <Step title="Tools bereitstellen">
    Nach der Verifizierung erstellt Revol automatisch **Agenten-Tools** basierend auf dem Integrationstyp. Zum Beispiel gibt die Verbindung von Gmail Ihren Agenten `send_email`, `read_inbox`, `search_emails` und andere Tools.
  </Step>
  <Step title="Dem Agenten zuweisen">
    Im **Tools**-Tab des Agenten aktivieren Sie die benoetigten Tools. Fuegen Sie sie dann zu Workflow-Knoten hinzu — der Agent kann diese Tools waehrend Konversationen ueber LLM-Funktionsaufrufe verwenden.
  </Step>
</Steps>

## Authentifizierungsmethoden

| Methode | Verwendet von | Funktionsweise |
|---|---|---|
| **API-Schluessel / Token** | Twilio, Ringostat, Binotel, Telegram | Zugangsdaten im Assistenten eingeben. Verschluesselt in der Datenbank gespeichert. |
| **Google OAuth 2.0** | Gmail, Docs, Sheets, Drive, Calendar, Meet, Google Ads | Klicken Sie auf "Mit Google verbinden", autorisieren Sie im Zustimmungsbildschirm. Revol speichert ein Refresh-Token und aktualisiert Access-Tokens automatisch. |
| **Meta OAuth** | WhatsApp, Instagram, Facebook Messenger, Facebook Ads | Klicken Sie auf "Mit Meta verbinden", autorisieren Sie ueber Meta. Revol tauscht den Code gegen ein langlebiges Token (~60 Tage). |
| **Webhook-URL** | Benutzerdefinierter Webhook | Keine Authentifizierung erforderlich — Sie geben die Ziel-URL und optionale Header an. |

## Kanaele

Ein **Kanal** ist die Verbindung zwischen einer Integration und einem bestimmten KI-Agenten. Eine Integration (z.B. Ihr Twilio-Konto) kann mehrere Agenten ueber separate Kanaele bedienen.

Verfuegbare Kanaltypen:

| Kanal | Erfordert Integration | Beschreibung |
|---|---|---|
| **Web-Widget** | Nein | Eingebautes Chat-Widget fuer Ihre Website |
| **Telefon** | VoIP-Integration | Ein-/ausgehende Sprachanrufe und SMS |
| **Telegram** | Telegram-Integration | Bot-Nachrichten in Telegram |
| **WhatsApp** | WhatsApp-Integration | WhatsApp Business-Nachrichten |
| **Instagram** | Instagram-Integration | Instagram Direct Messages |

Um einen Kanal einzurichten, gehen Sie zum **Kanaele**-Tab des Agenten, waehlen Sie den Kanaltyp und bestimmen Sie, welche Integration verwendet werden soll.

## Webhook-Sicherheit

Alle eingehenden Webhook-Endpunkte (VoIP-Anrufe, Messenger-Nachrichten) sind geschuetzt durch:

- **Signaturvalidierung** — kryptografische Verifizierung der Anfragenauthentizitaet pro Anbieter
- **Idempotenz** — Erkennung doppelter Webhooks zur Vermeidung von Doppelverarbeitung
- **Ratenbegrenzung** — 120 Anfragen pro Minute pro Kanal
- **IP-Zulassungsliste** — wo vom Anbieter unterstuetzt (z.B. Binotel)
