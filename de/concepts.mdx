## Unternehmen

Ein **Unternehmen** ist die zentrale Einheit in Revol. Alles — Produkte, Dateien, Agenten, Integrationen und Analysen — gehört zu einem Unternehmen.

Jedes Unternehmen hat:

- **Produkte & Dienstleistungen** — strukturierter Katalog mit Beschreibungen, Preisen, Fotos und Parametern
- **Dateimanager** — Dokumente, Links, Text, Video und Fotos für Wissensdatenbanken
- **KI-Agenten** — ein oder mehrere intelligente Assistenten, die mit Unternehmensdaten trainiert wurden
- **Integrationen** — verbundene Kanäle (Widget, VoIP, Messenger, Werbung, CRM)
- **Analysen** — Besucherverfolgung, Sitzungsdaten und Gesprächsverläufe

<Note>
  Ein Revol-Konto kann mehrere Unternehmen mit vollständig isolierten Daten verwalten — ideal für Agenturen oder Mehr-Marken-Unternehmen.
</Note>

## Produkte & Dienstleistungen

**Produkte** sind strukturierte Datenmodelle, die beschreiben, was Ihr Unternehmen anbietet. Im Gegensatz zu reinen Textdokumenten geben Produkte KI-Agenten präzise, abfragbare Informationen.

Jedes Produkt umfasst:

| Feld | Zweck |
|------|-------|
| **Name** | Produkt- oder Dienstleistungsbezeichnung |
| **Beschreibung** | Ausführlicher Text, den der Agent für Antworten verwendet |
| **Preis** | Preisinformationen |
| **URL** | Link zur Produktseite auf Ihrer Website |
| **Verfügbarkeit** | Ob das Produkt derzeit verfügbar ist |
| **Fotos** | Produktbilder, die der Agent teilen kann |
| **Parameter** | Strukturierte Schlüssel-Wert-Paare (Größe, Farbe, Dauer usw.) |

**Parameter** sind der leistungsstärkste Teil — strukturierte Eigenschaften, die es Agenten ermöglichen, exakte Antworten zu geben, anstatt unstrukturierten Text zu interpretieren. Siehe [Schnellstart → Produktparameter](/quickstart#product-parameters) für Beispiele.

## Wissensdatenbank

Die **Wissensdatenbank** ist der Inhalt, aus dem Ihre KI-Agenten lernen. Sie nutzt **RAG (Retrieval-Augmented Generation)** — Dokumente werden in Abschnitte aufgeteilt, in Vektor-Embeddings umgewandelt und während Gesprächen semantisch durchsucht.

### Inhaltstypen

| Typ | Formate | Funktionsweise |
|-----|---------|----------------|
| **Dokumente** | PDF, DOCX, TXT | Text wird extrahiert und in Abschnitte aufgeteilt |
| **Links** | Beliebige URL | Webseiteninhalt wird abgerufen und indexiert |
| **Text** | Freitext | Wird direkt als Wissensabschnitt hinzugefügt |
| **Video** | MP4, WebM, MOV | Audio wird über STT in Text transkribiert |
| **Fotos** | JPG, PNG | Text wird über OCR extrahiert |

### Zwei Wissensebenen

- **Dateien auf Unternehmensebene** — für alle Agenten im Unternehmen verfügbar
- **Dateien auf Produktebene** — an ein bestimmtes Produkt angehängt, werden verwendet, wenn der Agent über dieses Produkt spricht

## KI-Agenten

Ein **KI-Agent** ist ein intelligenter Assistent, der unabhängig mit eigener Persönlichkeit, Wissensdatenbank und eigenem Verhalten konfiguriert wird. Agenten können in drei Modi arbeiten:

| Modus | Beschreibung |
|-------|--------------|
| **Aktiv** | Die KI übernimmt vollständig die Kundenkommunikation — antwortet auf Nachrichten, führt Tool-Aufrufe durch, folgt dem Workflow |
| **Passiv** | Ihr Team übernimmt die gesamte Kommunikation. Der Agent arbeitet im Hintergrund — überwacht Gespräche, prüft die Einhaltung von Kommunikationsstandards und liefert Analysen |
| **Hybrid** | Eine Kombination: Die KI übernimmt Routineinteraktionen automatisch, während Ihr Team bei komplexen Fällen einspringt. Der Agent analysiert kontinuierlich alle Gespräche, unabhängig davon, wer antwortet |

<Tip>
  Der passive Modus ist ideal für Teams, die KI-gestützte Qualitätskontrolle wünschen, ohne menschliche Mitarbeiter zu ersetzen. Der Agent überwacht jedes Gespräch, prüft die Einhaltung von Skripten und Standards und liefert Erkenntnisse — ohne jemals direkt auf Kunden zu antworten.
</Tip>

### LLM-Anbieter

Jeder Agent (oder einzelne Workflow-Knoten) kann ein anderes KI-Modell verwenden:

| Anbieter | Standard-Modelle | Premium-Modelle |
|----------|-----------------|-----------------|
| **OpenAI** | GPT-4o-mini | GPT-4o |
| **Anthropic** | Claude 3.5 Haiku | Claude 3.5 Sonnet |
| **Google** | Gemini 2.0 Flash | Gemini 2.5 Pro |

<Info>
  Standard-Modelle sind schneller und verbrauchen Standard-Token. Premium-Modelle bieten höhere Qualität und verbrauchen Premium-Token.
</Info>

## Workflows

Ein **Workflow** ist ein visueller knotenbasierter Editor, der definiert, wie Ihr Agent Gespräche verarbeitet. Knoten sind durch Kanten verbunden, die den Datenfluss bestimmen.

### Knotentypen

| Knoten | Beschreibung |
|--------|--------------|
| **Start** | Einstiegspunkt — empfängt die Nachricht des Benutzers |
| **LLM** | Sendet Kontext an ein KI-Modell und erhält eine Antwort |
| **Bedingung** | Verzweigt den Ablauf basierend auf Regeln (Schlüsselwörter, Absicht, Variablen) |
| **Tool-Aufruf** | Führt eine externe Funktion aus (Produkte suchen, Verfügbarkeit prüfen) |
| **Spracheingabe (STT)** | Wandelt Sprache in Text für Sprachinteraktionen um |
| **Sprachausgabe (TTS)** | Wandelt Textantwort in Sprache um |
| **Antwort-Formatierer** | Kombiniert Ausgaben mehrerer Knoten zu einer einzigen Antwort |

### Standard-Workflow

Jeder neue Agent wird mit einem vorgefertigten Workflow geliefert:

```
Start → Product Agent  ─┐
     ├→ Media Agent    ─┤→ Response Formatter
     └→ Company Agent  ─┘
```

Drei spezialisierte Agenten arbeiten **parallel** — jeder bearbeitet seinen Bereich (Produkte, Medien, Unternehmensinformationen), und der Formatierer führt deren Ausgaben zusammen. Dies deckt die meisten Anwendungsfälle ohne jegliche Konfiguration ab.

Für komplexe Szenarien — bedingte Verzweigungen, benutzerdefinierte Tool-Aufrufe, mehrstufige Abläufe — verwenden Sie den [Workflow-Editor](/agents/workflow).

## Konversationen

Eine **Konversation** ist eine Chat-Sitzung zwischen einem Besucher und einem KI-Agenten. Alle Konversationen werden gespeichert und sind über die **Konversationen**-Seite zugänglich.

### Kanäle

Konversationen können über folgende Wege stattfinden:

- **Website-Widget** — eingebetteter Chat auf Ihrer Website
- **VoIP** — Telefonanrufe über Twilio, Ringostat oder Binotel
- **Messenger** — Telegram, WhatsApp Business, Instagram DM, Facebook Messenger
- **API** — programmatischer Zugriff

### Funktionen

- **Einheitlicher Posteingang** — alle Konversationen über alle Agenten und Kanäle an einem Ort
- **Filter** — nach Agent, Status, Datum, Kanal
- **Transkripte** — vollständiger Nachrichtenverlauf mit Zeitstempeln
- **Audiowiedergabe** — Anhören von Sprachanrufaufzeichnungen
- **Bewertungen** — Kundenzufriedenheitswerte pro Konversation

## Tracking

Der **Tracker** ist in dasselbe Widget-Skript integriert und überwacht automatisch die Besucheraktivität auf Ihrer Website.

### Was erfasst wird

| Daten | Beschreibung |
|-------|--------------|
| **Sitzungen** | Besucher-Browsing-Sitzungen — Gerät, Browser, Betriebssystem, Standort, Referrer |
| **Seitenaufrufe** | Einzelne Seitenbesuche innerhalb einer Sitzung |
| **Ereignisse** | Benutzerdefinierte Ereignisse — Klicks, Formularübermittlungen, Telefonanrufe |
| **Kampagnen** | UTM-basierte Gruppierung (Quelle, Medium, Kampagne, Begriff, Inhalt) |

### Automatisierung

- **Trigger** — automatisierte Aktionen basierend auf Besucherverhalten (Verweildauer, Scrolltiefe, UTM-Übereinstimmung, Gerätetyp). Können Chat-Overlays oder benutzerdefinierte Nachrichten anzeigen.
- **Rufnummerntausch** — dynamischer Telefonnummernaustausch für Anrufverfolgung und Zuordnung.

## Integrationen

**Integrationen** verbinden Ihre KI-Agenten mit externen Diensten und Kommunikationskanälen.

| Kategorie | Anbieter | Funktionalität |
|-----------|----------|----------------|
| **VoIP** | Twilio, Ringostat, Binotel | Telefonanrufe — Agent nimmt eingehende Anrufe entgegen |
| **Messenger** | Telegram, WhatsApp, Instagram, Facebook Messenger | Chat über Messaging-Apps |
| **Google Workspace** | Gmail, Docs, Sheets, Drive, Calendar, Meet | Lesen und Interagieren mit Google-Diensten |
| **Werbung** | Google Ads, Facebook Ads | Kampagnenleistung anzeigen und Anzeigen verwalten |
| **Webhook** | Beliebiger HTTP-Endpunkt | Lead-Daten, Ereignisse oder Benachrichtigungen an CRM oder andere Systeme senden |

Jede Integration wird pro Unternehmen konfiguriert und kann mit einem oder mehreren Agenten verbunden werden.

### Was als Token zählt

Jede KI-Interaktion besteht aus zwei Teilen:

- **Eingabe-Token** — Ihre Nachricht + System-Prompt + abgerufene Wissensdatenbank-Abschnitte + Gesprächsverlauf
- **Ausgabe-Token** — die Antwort des Agenten

Verschiedene Modelle verbrauchen unterschiedliche Token-Typen:

| Token-Typ | Modelle | Pläne |
|-----------|---------|-------|
| **Standard** | GPT-4o-mini, Claude 3.5 Haiku, Gemini 2.0 Flash, Llama 3.1 8B | Alle Pläne |
| **Premium** | GPT-4o, Claude 3.5 Sonnet, Gemini 2.5 Pro, Llama 3.3 70B | Premium+ |

### Was passiert, wenn Sie das Limit erreichen

| Ressource | Verhalten |
|-----------|-----------|
| Standard- / Premium-Token | Agent antwortet nicht mehr bis zum nächsten Abrechnungszeitraum oder Paketkauf |
| Speicher | Neue Dateien können nicht hochgeladen werden |
| Sprache | Sprachfunktionen werden deaktiviert |
| Konversationen | Neue Konversationen werden blockiert |

Revol sendet Benachrichtigungen bei **80 %** (Warnung) und **90 %** (kritisch) Nutzung, damit Sie handeln können, bevor das Limit erreicht wird.
