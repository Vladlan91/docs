## Überblick

Tool-Protokolle bieten vollständige Transparenz darüber, was Ihr KI-Agent während Konversationen tut — jeder Tool-Aufruf, jede RAG-Suche und jedes externe Kanalereignis wird mit Ein-/Ausgabedaten, Ausführungszeit und Status aufgezeichnet.

Öffnen Sie die Tool-Protokolle, indem Sie auf die Schaltfläche **Tool-Protokolle** in der oberen rechten Ecke des Agenten-Editor-Modals klicken.

## Zwei Tabs

Das Modal hat zwei Tabs:

### Tool-Protokolle

Zeichnet jede Tool-Ausführung während KI-Konversationen auf. Jeder Eintrag zeigt:

| Feld | Beschreibung |
|-------|-------------|
| **Tool-Name** | Die aufgerufene Funktion (z.B. `rag_search`, `get_products`, `get_company_info`) |
| **Knotenname** | Welcher Workflow-Knoten dieses Tool ausgelöst hat |
| **Knotentyp** | Knotenkategorie — Orchestrator, Produkt-Agent, Medien-Agent, Firmen-Agent, Formatierer oder Benutzerdefiniert |
| **Status** | `success`, `error` oder `no_results` |
| **Ausführungszeit** | Dauer in Millisekunden |
| **Datum** | Wann das Tool ausgeführt wurde |

Klicken Sie auf einen beliebigen Protokolleintrag, um ihn aufzuklappen und die vollständige **Eingabe** (Anfrageparameter als JSON) und **Ausgabe** (Antwortdaten als JSON) zu sehen. Wenn der Status `error` ist, wird die Fehlermeldung oben im aufgeklappten Bereich angezeigt.

<Tip>
  Tool-Protokolle sind nach **Runden** gruppiert. Eine Runde stellt eine LLM-Function-Calling-Iteration dar — die KI kann mehrere Tools in einer einzigen Runde aufrufen, bevor sie eine Antwort generiert. Rundentrennlinien helfen Ihnen, die Reihenfolge der Tool-Aufrufe innerhalb eines Konversationsdurchlaufs zu verstehen.
</Tip>

### Kanal-Protokolle

Zeichnet Ereignisse von externen Kommunikationskanälen auf — Webhooks, Telefonanrufe, Messenger-Nachrichten. Jeder Eintrag zeigt:

| Feld | Beschreibung |
|-------|-------------|
| **Kanaltyp** | Widget, Telefon, Telegram, WhatsApp, Instagram, etc. |
| **Richtung** | `inbound` (empfangen) oder `outbound` (gesendet) |
| **Externe ID** | Anrufer- oder Absenderkennung vom externen Anbieter |
| **Status** | `success` oder `error` |
| **Datum** | Wann das Ereignis aufgetreten ist |

Klicken Sie zum Aufklappen, um den vollständigen **Anfrage-Payload** (rohe Webhook-Daten) und die **Antwort** (was zurückgesendet wurde) anzuzeigen.

## Was protokolliert wird

Tool-Protokolle werden automatisch erstellt, wenn der Agent:

- **Die Wissensdatenbank durchsucht** — `rag_search` mit der Abfrage und Anzahl der gefundenen Ergebnisse
- **Ein Funktions-Tool aufruft** — jedes im Toolset des Agenten konfigurierte Tool (z.B. `get_products`, `get_documents`, `get_photos`, `get_videos`, `get_company_info`)
- **Integrations-Tools ausführt** — Google Ads, Facebook Ads und andere verbundene Dienst-Tools

Kanal-Protokolle werden erstellt, wenn:

- **Ein eingehender Webhook eintrifft** — Telefonanruf, Messenger-Nachricht oder ein anderes externes Kanalereignis
- **Eine ausgehende Nachricht gesendet wird** — die Antwort des Agenten, die über einen Kanal zugestellt wird

## Protokolle löschen

Verwenden Sie die Schaltfläche **Protokolle löschen** (Papierkorb-Symbol) oben im Modal, um alle Protokolle für den aktuellen Agenten zu löschen. Diese Aktion ist unwiderruflich.

<Note>
  Protokolle werden in einer partitionierten Datenbanktabelle für optimale Leistung gespeichert. Agenten mit hohem Volumen können Tausende von Einträgen ansammeln — verwenden Sie „Protokolle löschen" regelmäßig, um die Benutzeroberfläche reaktionsschnell zu halten.
</Note>
