## Ueberblick

Der Revol Tracker ueberwacht die Besucheraktivitaet auf Ihrer Website. Er erfasst Sitzungsdaten, Seitenaufrufe, benutzerdefinierte Ereignisse und Kampagnenzuordnung — alles ueber dasselbe Skript, das auch das Chat-Widget betreibt. Kein zusaetzlicher Code erforderlich.

Das Dashboard bietet einen Echtzeit-Ueberblick ueber den Traffic und das Besucherverhalten auf Ihrer Website. Die Daten werden automatisch alle **60 Sekunden** aktualisiert.

Analysedaten werden in monatlichen Partitionen gespeichert und basierend auf Ihrem Tarif automatisch bereinigt:

| Tarif | Datenaufbewahrung |
|---|---|
| Free | 3 Monate |
| Premium | 12 Monate |

### Filter

Verwenden Sie Filter, um die Dashboard-Daten einzugrenzen:

**Schnelle Periodenauswahl:**

| Zeitraum | Beschreibung |
|---|---|
| **24h** | Letzte 24 Stunden |
| **7d** | Letzte 7 Tage (Standard) |
| **30d** | Letzte 30 Tage |
| **90d** | Letzte 90 Tage |

**Erweiterte Filter** (Dropdown):

| Filter | Beschreibung |
|---|---|
| **Geraet** | Desktop, Mobil oder Tablet |
| **Land** | Nach Land filtern (2-Buchstaben-ISO-Code) |
| **Quelle** | UTM-Quelle (z.B. google, facebook) |
| **Seiten-URL** | Nach bestimmter Seite filtern |
| **Browser** | Chrome, Firefox, Safari, usw. |
| **Betriebssystem** | Windows, macOS, iOS, Android, usw. |
| **Absprung** | Nur abgesprungene oder nicht abgesprungene Sitzungen anzeigen |
| **Kampagne** | Aus Ihrer Kampagnenliste auswaehlen |
| **Medium** | cpc, organic, social, email, referral |
| **Datum von** | Benutzerdefiniertes Startdatum |
| **Datum bis** | Benutzerdefiniertes Enddatum |

<Tip>
  Ohne Filter verwendet das Dashboard voraggregierte Daten fuer schnelles Laden. Bei angewendeten Filtern wird auf Rohabfragen umgeschaltet, um genaue Ergebnisse zu liefern — dies kann etwas laenger dauern.
</Tip>

### Sitzungen

Gesamtanzahl der Besuchersitzungen im ausgewaehlten Zeitraum. Eine Sitzung beginnt, wenn ein Besucher Ihre Website laedt, und endet nach 30 Minuten Inaktivitaet oder wenn er die Seite verlaesst.

**Berechnung:** `COUNT(*)` aller Sitzungen im Zeitraum.

### Einzelbesucher

Anzahl der eindeutigen Besucher, identifiziert durch ein einzigartiges Browser-Token. Eine Person, die Ihre Website 5-mal besucht = 1 Einzelbesucher, aber 5 Sitzungen.

**Berechnung:** `COUNT(DISTINCT guest_token)` aus Sitzungen.

### Absprungrate

Prozentsatz der Sitzungen, in denen der Besucher nur eine Seite angesehen und ohne weitere Interaktion verlassen hat. Eine niedrigere Absprungrate deutet im Allgemeinen auf besseres Engagement hin.

**Berechnung:** `(Sitzungen mit 1 Seitenaufruf / Gesamtsitzungen) × 100%`

<Note>
  Der Trendpfeil fuer die Absprungrate ist **invertiert** — ein Rueckgang (gruener Pfeil) bedeutet Verbesserung, waehrend ein Anstieg (roter Pfeil) bedeutet, dass mehr Besucher nach einer Seite abspringen.
</Note>

### Durchschn. Dauer

Durchschnittliche Zeit, die Besucher pro Sitzung auf Ihrer Website verbringen, angezeigt in Sekunden oder Minuten.

**Berechnung:** `AVG(duration_seconds)` ueber alle Sitzungen.

### Seiten / Besuch

Durchschnittliche Anzahl der pro Sitzung angesehenen Seiten. Hoehere Werte deuten darauf hin, dass Besucher mehr Ihrer Inhalte erkunden.

**Berechnung:** `AVG(pages_visited)` ueber alle Sitzungen.

### Conversions

Anzahl der eindeutigen Sitzungen, die mindestens ein als **Conversion-Ziel** markiertes Ereignis ausgeloest haben. Sie definieren im Ereignisse-Tab, welche Ereignisse als Conversions zaehlen.

**Berechnung:** `COUNT(DISTINCT session_id)`, wobei die Sitzung mindestens ein Ereignis mit `is_conversion = true` enthaelt.

<Tip>
  Um Conversions einzurichten, gehen Sie zum **Ereignisse**-Tab, erstellen oder bearbeiten Sie ein Ereignis und aktivieren Sie den Schalter **"Conversion-Ziel"**. Jeder Ereignistyp kann eine Conversion sein — Formulareinreichungen, Schaltflaechenklicks, Seitenaufrufe oder benutzerdefinierte Ereignisse.
</Tip>

## Neue vs. wiederkehrende Besucher

Ein Ringdiagramm, das die Aufteilung zeigt zwischen:

- **Neue Besucher** — Erstbesucher (nur 1 Sitzung insgesamt)
- **Wiederkehrende Besucher** — Besucher, die bereits auf Ihrer Website waren (2+ Sitzungen)

Eine gesunde Mischung haengt von Ihrem Geschaeft ab: Inhaltsseiten streben mehr wiederkehrende Besucher an, waehrend werbegetriebene Landingpages ueberwiegend neue Besucher erwarten.

| Gruppe | Beschreibung |
|---|---|

## Scrolltiefe

Ein Histogramm, das zeigt, wie weit Besucher auf Ihren Seiten scrollen, in 4 Gruppen:

| Gruppe | Beschreibung |
|---|---|
| **0–25%** | Hat nur den oberen Teil der Seite gesehen |
| **25–50%** | Bis zur Mitte gescrollt |
| **50–75%** | Den groessten Teil der Seite gelesen |
| **75–100%** | Das Ende erreicht |

Nutzen Sie dies, um zu verstehen, ob Besucher Ihren gesamten Inhalt lesen oder fruehzeitig abbrechen.
