## Przegląd

Logi narzędzi zapewniają pełną widoczność tego, co Twój agent AI robi podczas rozmów — każde wywołanie narzędzia, wyszukiwanie RAG i zdarzenie zewnętrznego kanału jest rejestrowane z danymi wejściowymi/wyjściowymi, czasem wykonania i statusem.

Otwórz Logi narzędzi, klikając przycisk **Logi narzędzi** w prawym górnym rogu okna modalnego edytora agenta.

## Dwie zakładki

Okno modalne ma dwie zakładki:

### Logi narzędzi

Rejestruje każde wykonanie narzędzia podczas rozmów AI. Każdy wpis pokazuje:

| Pole | Opis |
|------|------|
| **Nazwa narzędzia** | Funkcja, która została wywołana (np. `rag_search`, `get_products`, `get_company_info`) |
| **Nazwa węzła** | Który węzeł przepływu pracy wyzwolił to narzędzie |
| **Typ węzła** | Kategoria węzła — Orkiestrator, Agent produktowy, Agent multimedialny, Agent firmowy, Formater lub Niestandardowy |
| **Status** | `success`, `error` lub `no_results` |
| **Czas wykonania** | Czas trwania w milisekundach |
| **Data** | Kiedy narzędzie zostało wykonane |

Kliknij dowolny wpis logu, aby go rozwinąć i zobaczyć pełne **Wejście** (parametry żądania jako JSON) i **Wyjście** (dane odpowiedzi jako JSON). Jeśli status to `error`, komunikat o błędzie jest wyświetlany na górze rozwiniętej sekcji.

<Tip>
  Logi narzędzi są pogrupowane w **rundy**. Runda reprezentuje jedną iterację wywoływania funkcji przez LLM — AI może wywoływać wiele narzędzi w jednej rundzie przed wygenerowaniem odpowiedzi. Separatory rund pomagają zrozumieć sekwencję wywołań narzędzi w ramach jednej tury konwersacji.
</Tip>

### Logi kanałów

Rejestruje zdarzenia z zewnętrznych kanałów komunikacji — webhooków, połączeń telefonicznych, wiadomości z komunikatorów. Każdy wpis pokazuje:

| Pole | Opis |
|------|------|
| **Typ kanału** | Widżet, Telefon, Telegram, WhatsApp, Instagram itp. |
| **Kierunek** | `inbound` (odebrane) lub `outbound` (wysłane) |
| **Zewnętrzne ID** | Identyfikator dzwoniącego lub nadawcy od zewnętrznego dostawcy |
| **Status** | `success` lub `error` |
| **Data** | Kiedy zdarzenie wystąpiło |

Kliknij, aby rozwinąć i zobaczyć pełny **Ładunek żądania** (surowe dane webhooka) i **Odpowiedź** (co zostało wysłane z powrotem).

## Co jest logowane

Logi narzędzi są tworzone automatycznie, gdy agent:

- **Przeszukuje bazę wiedzy** — `rag_search` z zapytaniem i liczbą znalezionych wyników
- **Wywołuje narzędzie funkcyjne** — dowolne narzędzie skonfigurowane w zestawie narzędzi agenta (np. `get_products`, `get_documents`, `get_photos`, `get_videos`, `get_company_info`)
- **Wykonuje narzędzia integracyjne** — Google Ads, Facebook Ads i inne narzędzia podłączonych usług

Logi kanałów są tworzone, gdy:

- **Przychodzi webhook** — połączenie telefoniczne, wiadomość z komunikatora lub dowolne zdarzenie kanału zewnętrznego
- **Wysyłana jest wiadomość wychodząca** — odpowiedź agenta dostarczona przez kanał

## Czyszczenie logów

Użyj przycisku **Wyczyść logi** (ikona kosza) na górze okna modalnego, aby usunąć wszystkie logi dla bieżącego agenta. Ta operacja jest nieodwracalna.

<Note>
  Logi są przechowywane w partycjonowanej tabeli bazy danych dla wydajności. Agenci o dużym ruchu mogą gromadzić tysiące wpisów — regularnie używaj opcji Wyczyść logi, aby interfejs pozostawał responsywny.
</Note>
